This post is giving you the details of research, purpose, and key features in Internet Key exchange (IKE). <\/p>\n\n\n\n

Overview<\/h1>\n\n\n\n
    \n
  • In computing, Internet Key Exchange<\/strong> (IKE<\/strong>, sometimes IKEv1<\/strong> or IKEv2<\/strong>, depending on version) is the protocol used to set up a security association (SA) in the IPsec protocol suite.<\/li>\n<\/ul>\n\n\n\n

    Key features<\/h2>\n\n\n\n
      \n
    • Auto create a new key<\/li>\n\n\n\n
    • Protecting on attacks which block the circuit like DDos (Denial-of-service attack)<\/li>\n\n\n\n
    • Using digital key<\/li>\n\n\n\n
    • Using the same key<\/li>\n\n\n\n
    • Providing means for both partners to agree on protocols, algorithms, and keys to use.<\/li>\n\n\n\n
    • Make sure the key exchange goes to the right user<\/li>\n\n\n\n
    • Managing keys after being accpeted<\/li>\n\n\n\n
    • Allow dynamic endorsement between peer objects.<\/li>\n<\/ul>\n\n\n\n

      Purpose<\/h2>\n\n\n\n
        \n
      • Internet key exchanged uses to set up a secure, authenticated communications channel between two devices.<\/li>\n\n\n\n
      • During key exchange, IKE uses a symmetric encryption algorithm, these keys will be changed over time -> limit the cracking status of attackers<\/li>\n<\/ul>\n\n\n\n
